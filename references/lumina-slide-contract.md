# Lumina Slide Contract

Generated slides must satisfy this contract before they are sent to `POST /api/agent/presentations`.

## Slide Object

```json
{
  "title": "Slide title",
  "slide_type": "title|content|list|two_column|comparison|quote|timeline|stats|end",
  "html": "<div class=\"slide\" style=\"width:1920px;height:1080px;...\">...</div>",
  "css": "",
  "speaker_notes": "Optional speaker notes"
}
```

## HTML Requirements

- Root element: `<div class="slide">`.
- Dimensions: `width: 1920px; height: 1080px;`.
- Root overflow: `overflow: hidden`.
- Content containers: avoid clipping; prefer `overflow: visible`, flex/grid layout, and responsive text wrapping.
- Use `contenteditable="true"` on editable text.
- Use stable unique `data-field="..."` attributes on editable text.
- Keep `data-editable="true"` if generated by a Slaydplus prompt, but do not rely on it alone.
- Avoid external JavaScript.
- Avoid secrets, private URLs, and credentials.

## Editable Text Example

```html
<h1
  contenteditable="true"
  data-editable="true"
  data-field="slide_1_title"
  style="font-size: 88px; line-height: 1.05;"
>
  AI Sales Assistant
</h1>
```

## CSS Strategy

Prefer inline styles for portability. Use global `theme_css` for shared fonts, resets, and CSS variables. Use per-slide `css` only when it makes the payload easier to maintain.

## Before Publishing

Check every slide for:

- No clipped text.
- Good contrast.
- Consistent theme.
- One clear message.
- Editable text fields.
- Valid hosted image URLs if images are used.
